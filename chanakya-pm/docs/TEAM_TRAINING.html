1  Signing in

You have two ways to sign in. Google is preferred — it's the fastest and most secure.

Option A — Sign in with Google (recommended)

  1. Open https://chanakya-pm.onrender.com
  2. Click Sign in with Google
  3. Pick your @whitelotusgroup.in work account
  4. You're in.

First-time Google sign-in from @whitelotusgroup.in auto-creates an account with edit permission. Need elevated access? Ping Vandana.

Option B — Email and password

If your account was pre-seeded with a password:

  1. Enter your work email
  2. Enter the shared default password (ask Vandana)
  3. Click Sign In

Forgot your password? Email Vandana. Self-reset isn't built in yet.

Session length

  • You stay signed in for 7 days
  • A warning banner appears 15 minutes before expiry so you can save and sign in again without losing work
  • If the session expires mid-edit, you'll see "Your session has expired" and be returned to the login screen

2  First two minutes in the app

After signing in, you land on the Select Development screen.

  • Amanvana and Kandhavara are the two main developments
  • Click Open Dashboard → on the one you work with
  • Or click View All Developments to see everything

Inside a development you'll see villa cards. Click any villa to open its dashboard.

Tip. The badge next to the Chanakya logo (top-left) shows which development you're in. Click it to go back to the Select Development screen.

4  Common tasks — step by step

Add a new task

Requires edit permission

  1. Open any villa → WBS tab
  2. Click + Add Task (top-right of the WBS toolbar)
  3. Pick task type: group task or subtask
  4. Fill in name, dates, phase, status, progress
  5. Optionally add predecessors, assignee, cost, notes
  6. Click Save

The task auto-saves to the server within 1.5 seconds. Other team members will see it on their next refresh.

Edit a task

  1. Click the task row in the WBS tab
  2. The task panel opens on the right
  3. Edit any field → changes save automatically
  4. Or click the task name to open the full edit modal

Quick edits from the Gantt:

  • Drag a task bar left or right to shift its dates (preserves duration)
  • A toast confirms the new dates with an Undo button

Mark a task complete, in-progress, or blocked

  1. Click the task
  2. Change the Status dropdown
  3. If status is blocked or on-hold, a "Blocking reason" field appears — fill it in so teammates know why

Delete a task

  1. Click the task → Delete button
  2. A toast appears with an Undo button (6-second window)
  3. After 6 seconds the delete is permanent (but still recoverable via the audit log for admins)

Capture a baseline

Requires edit or manage

  1. Open a villa → Baselines tab
  2. Click Capture New Baseline
  3. Give it a name ("Initial Schedule", "Revised Plan v2")
  4. Optionally add notes
  5. Click Save

Baselines are read-only snapshots. Compare any two later to see what changed.

Import tasks from CSV / Excel

Requires manage permission

  1. Click Import (top-right, visible only if you have manage)
  2. Choose By Phase, By Villa, or WL Master Schedule mode
  3. Upload the file
  4. Map columns if prompted
  5. Preview → Import

Export data

Requires manage permission

  1. Click Export (top-right)
  2. A JSON backup file downloads
  3. Every export is logged to the audit trail automatically

Request or approve an approval

  1. Any user can request an approval from a task's panel
  2. Users with approve permission (Pavan, Chetan, Vandana) see the Approvals tab with pending items
  3. Approve or reject with a comment

View the audit log

Requires manage permission

  1. Click the 📋 Audit Log button (top-right)
  2. See the last 500 actions: who changed what, when

6  Working together

Auto-save and conflicts

  • Every change you make auto-saves every ~1.5 seconds
  • Other team members see your changes on reload or next save
  • If two people edit the same villa at the same time, whoever saves second sees: "Another user saved changes after you last synced — reload to see the latest, then retry your edit"
  • Click Reload — your unsaved change needs to be redone, but nobody's work is silently lost

Offline or bad network

  • If the connection drops, a yellow banner appears: "Your changes aren't syncing to the server. Retrying…"
  • The app retries automatically every 5 seconds
  • If saves keep failing for 30+ seconds, the banner turns red and asks you to stop editing until the connection returns
  • Click Retry now to force a retry, or Reload to start fresh

What's logged

Every create, edit, delete, reschedule, import, export, and approval is logged to the audit trail with your name and timestamp. Admins can review the log at any time via the 📋 Audit Log button.

9  Troubleshooting

"I can't log in"

  • Double-check you're at https://chanakya-pm.onrender.com
  • Try Google sign-in if password fails
  • If you see "Chanakya couldn't finish loading", click Reload app. If it keeps happening, send a screenshot to Vandana.

"Everything looks broken / raw code on screen"

Hard-reload (Cmd+Shift+R on Mac, Ctrl+Shift+R on Windows). That clears the cached version. If still broken, report to Vandana.

"The screen keeps saying 'Another user saved'"

Someone else is editing the same villa. Click Reload, then redo your edit. If this happens repeatedly, coordinate with the other person in Slack to avoid stepping on each other's changes.

"I don't see the ⚙ Manage / Import / + New Villa buttons"

You don't have manage permission. That's expected for most users. Ask Vandana if you need it.

"Gantt chart is slow"

Try switching the zoom from Week to Month or Quarter — fewer columns means faster rendering.

"I accidentally deleted something"

If within the last 6 seconds, the toast has an Undo button. If longer, ping Vandana — she can restore from the audit log.

10  Known limitations

These are on the roadmap but not built yet:

  • Mobile and tablet layout — desktop / laptop only for now
  • Password self-reset — email Vandana if you forget yours
  • Photo / document attachments on tasks — use shared drive or email for now
  • Realtime live updates — you see other people's changes on reload or next save, not instantly. Usually no more than a couple of seconds' delay.
  • Bulk task editing — one task at a time for most fields (except Gantt drag for date shifts)
